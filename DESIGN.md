## 1. Design direction

**Dark-first, dense, branded, financial-services.** An operator's cockpit, not airy
SaaS. A **dark navy shell** (sidebar + topbar) wraps a **light content canvas** — that
high-contrast chrome-vs-content split is the core visual move. Three signature markers
carry identity and must never be lost:

1. **Dark navy shell** with light canvas.
2. **Uppercase DM Mono section labels** (11px, tracked) — the strongest character marker.
3. **Identity lockup** — the "M / Markist / … COMMAND CENTER" block atop every sidebar.

Color strategy: **Restrained** (Product floor) — tinted neutrals + one indigo accent,
with a disciplined semantic-status vocabulary and a single signature gold.

All tokens are HSL CSS variables in `src/app/globals.css :root`, surfaced through
`tailwind.config.ts`. **Never hardcode a color — always resolve through a token.**

### 2.3 Signature gold & semantics

- **Gold** (`--gold`, `--gold-deep` text-grade) = attention + **assumption** only.
  Never success. `--gold-deep` (`38 76% 34%`) is the ≥4.5:1 text-grade tint.
- **Purple** (`--status-security` / `--status-escalated` `262 47% 42%`) = securities /
  FFS-managed / escalated. Makes the firewall visible.
- **Green** (`--status-won` `152 55% 39%`) = won / placed / active only.
- **Red** (`--status-lost` `0 72% 51%`) = lost / blocked / error only.

Status ramp: `--status-{draft,active,pending,won,lost,blocked,escalated,assumption,security}`,
each surfaced as `bg-status-*` / `text-status-*` and consumed by `<Badge>`.

### 2.4 Type, space, radius, elevation, motion

- **Fonts:** DM Sans (body/UI), DM Mono (labels, numerics) via `next/font`. Exposed as
  `font-sans` / `font-mono`. **Fixed rem scale, not fluid** (Product register).
- **Type scale:** display 30/36 · h1 24/32 · h2 18/26 · h3 15/22 · body 14/21 ·
  small 13/18 · label 11/16 (DM Mono, `.mono-label`) · numeric (DM Mono, `.numeric`,
  tabular-nums). **Every money value, policy #, date, ID, % renders in `.numeric`.**
- **Spacing:** 4 / 8 / 12 / 16 / 24 / 32 / 48. Section gap 24, card padding 16.
- **Radius:** `--radius: 0.625rem` → `rounded-sm/md/lg/xl` derived. No oversized radii.
- **Elevation:** layered navy-tinted `--shadow-xs…xl` → `shadow-elev-*`. Cards sit flush
  (`shadow-elev-xs`), lift on interaction, overlays float on `xl`. No heavy gray drops.
- **Motion:** 150–250ms, ease-out. `fade-in-up`, `shimmer`, accordion keyframes only.
  Motion conveys state, not decoration. Global `prefers-reduced-motion` reset in place.
- **Density:** table rows 40px, header 32px. Sidebar 260px, topbar 56px, content
  max-width 1400px.

## 4. Archetype shells (`src/components/archetypes`)

Reusable, Server-Component-safe page skeletons. **Compose pages from these — do not
hand-roll page chrome.**

- **Chrome:** `PageHeader` (title + description + breadcrumb + actions), `Breadcrumb`.
- **A1 Dashboard:** `DashboardShell`, `StatTile` (KPI tile, links to source, hover accent).
- **A2 List:** `ListShell` (+ toolbar slot).
- **A3 Detail:** `DetailShell` (navy header band, status chips, related-records rail).
- **A4 Board:** `BoardShell`, `BoardColumn` (mono header + count + value total).
- **A5 Form:** `FormShell` (max-w-2xl, sticky footer). **A6 Wizard:** `WizardShell`.
- **A10 Settings:** `SettingsShell`, `SettingsSection`.
- **A11 Report:** `ReportShell`. **A12 Integration:** `IntegrationShell` (fallback note).
- **A13 Auth:** `AuthShell` (branded navy backdrop + identity lockup).
- **States (every page ships all):** `EmptyState`, `ForbiddenState`, `ListSkeleton`,
  `CardsSkeleton`, `ErrorState` (client), `StatusBadge`, `AssumptionBadge`.

## 5. The branded shell (`src/components/portal`)

One `PortalShell` powers all six authenticated portals (`portalLabel`, `nav`, `banner`,
`panels`, and topbar wiring props). Topbar: sticky 56px navy, global search, AI, bell,
`ProfileMenu`. Sidebar: 260px `shell-gradient`, `IdentityLockup`, nav grouped by OS
cluster under mono labels, optional character panels (FSA: AI Live Status, GDC Tier,
FFS Contacts). Mobile: `MobileTabBar`. Nav items via `NavLink` (36px, active =
raised + 2px accent bar + weight bump).

---

## 6. Applying it (the Definition of Done every page inherits)

Wired real data · Zod-validated inputs · enforced permissions (403 via `ForbiddenState`)
· empty / loading (skeleton, never bare spinner) / error (isolated, retryable) / success
states · responsive desktop→tablet→mobile · a11y (labels, keyboard, aria, AA contrast on
both shell and canvas) · audit events written · gold assumption badge on every config
default · purple FFS-managed marker on every `is_security` row · no dead ends.
