# FSOS Design Audit — Impeccable

> Full-surface design audit across 241 pages / 7 portals / 125 components. Generated by
> the Impeccable `audit` workflow (parallel sweeps: shared UI foundation, token
> discipline, cross-portal consistency, flagship dashboard). This is the backlog that
> drives the continuous refinement loop; items are ordered by **leverage** (fix-once →
> propagates-to-N-pages) not by portal.

## Headline

The authenticated app is **already token-disciplined and archetype-driven** — all six
portals share one `PortalShell`, no page hand-rolls its title bar, no arbitrary z-index,
no raw palette colors in core app routes. This is a mature system. The real debt is in
three concentrated bands:

1. **Shared-primitive state gaps** (Button has no loading state; nav has no focus ring) —
   fixing these propagates to every page.
2. **Cross-portal divergence** (only FSA has nav icons/grouping + wired topbar; 3 stub
   dashboards; dead search/bell controls in 5 portals).
3. **The public/unauthenticated surface** bypasses the token system entirely (hardcoded
   hex, raw palette classes, inline styles, emoji icons).

## Band 3 — Public/unauthenticated surface (token bypass)

Worst-offending cluster (predates the token system): `app/[slug]/page.tsx`,
`app/upload/[slug]/page.tsx`, `app/about/page.tsx`, `components/pages/ClientFormPortal.tsx`,
`EventsIndex.tsx`, `Unsubscribe.tsx`, `WorkshopRegister.tsx`, `components/public/PublicForm.tsx`,
`WorkshopRegisterForm.tsx`, `terms/page.tsx`, `privacy/page.tsx`.

- ~153 hardcoded hex, ~190 inline `style={{}}`, 59 raw palette classes, ~19 emoji icons.
- **Fix:** migrate to tokens + `ui/*` primitives + lucide. Legacy Command Center files
  (`fsos_command_center.jsx`, `fsos_forms_system.jsx`, `CommandCenter.tsx`) are
  intentionally inline-styled per CLAUDE.md §1.6 — **excluded**. Status: **backlog**
  (large mechanical sweep; scheduled after foundation lands).

## Band 4 — Flagship dashboard (`(fsa)/app/page.tsx`)

Today the FSA home is one flat 12-tile personalized KPI grid (`DashboardGrid`) with a
strong gold **triage** mechanic and a one-line "N queues need action" summary. Rich data
(pipeline-by-stage, commission-by-agency, briefing "what to do next", compliance events)
is stranded in the `/executive/*` back-rooms.

- **Preserve:** the personalized drag/resize/save widget grid (real feature), and the
  gold-attention triage engine (`isAttentionWidget`) — do not remove.
- **This iteration:** promote triage from a one-line summary to a prominent server-rendered
  **"Needs you today"** band above the grid, so the home leads with *what needs a decision*.
  Uses existing attention metrics; degrades independently. Status: **fixed**.
- **Backlog:** period/trend deltas in `metrics.ts` (Δ vs prior period); inline the
  pipeline-by-stage + commission-by-agency tables under banded sections.

## Continuous loop

Impeccable is applied iteratively, not once. Each iteration: re-audit changed surfaces →
fix highest-leverage remaining item → re-verify (`npm run build` + a11y/contrast) → update
this backlog. Iteration 1 (this pass) clears Band 1 in full, most of Band 2, and the
dashboard triage band. Bands 2 (P4–P6), 3, and 4-backlog remain for subsequent passes.
