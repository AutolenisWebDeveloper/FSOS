Co-worker Session Best Practices
1
One task per session
Never ask Claude to build multiple things in one session. "Fix the build error" is a session. "Deploy to Vercel" is a session. Scope is everything.
2
Always paste current build status
Start every session with the exact error output from your last npm run build. If it's clean, say "build is passing clean." This prevents duplicate work.
3
Never skip phases
The implementation guide has a phase order for a reason. Data before scoring. Scoring before GHL. GHL before AI agents. Skipping phases creates dependencies that break downstream.
4
Verify before moving on
Every session ends with a verification step. Build passes, page loads, record appears in Supabase. Do not start Session 4 until Session 3 is verified.
5
Credentials only when needed
Do not provide Supabase or GHL credentials until Session 3. The build and integration work does not need them. Provide credentials exactly when asked, phase by phase.

Educate clients about financial concepts

Pre-Launch Compliance Checklist
1
Confirm TRAIGA disclosure language
Ensure the TRAIGA AI disclosure language for SMS and voice is in place before activating AI agents. The FSA owns this; FFS Compliance TX (253) 242-0597 can verify wording if you want. Keep a written record.
2
Confirm TCPA consent mechanism
Every contact who receives automated SMS must have given written prior express consent. Verify your consent capture is storing to the consent_ledger table in Supabase.
3
Write your AIS Program document
Texas TRAIGA 2026 safe harbor requires a written AI governance program. Describes what AI tools you use, their purpose, and your oversight process. 2-3 pages is sufficient.
4
Confirm FNA output is educational-only
Before sending any FNA report to a client, confirm it qualifies as educational, not investment advice (guardrail §4.2). The FSA owns this review; FFS Compliance TX (253) 242-0597 is available for a second read if you want one.
