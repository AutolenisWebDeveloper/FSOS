Phase 01
Supabase Schema
Status: BUILT — needs to be executed · Supabase: ynxaqeejjmeilpwmuuie
The complete schema SQL is in the repo. It has never been run against your live Supabase project. Running it creates all 17 tables, scoring functions, RLS policies, pg_cron schedule, and seeds your 4 agency owners and 7 commission products in one pass.
Phases 1 and 2 are executed together — the scoring functions and pg_cron schedule are inside the same SQL file. Running Phase 1 activates Phase 2 simultaneously.
Phase 02
Scoring Engine
Status: BUILT — activated when Phase 1 SQL runs · Runs nightly 2AM CT
Five scoring functions (OPRA, Conversion, Life, Retirement, Business) plus run_nightly_scoring() and calculate_case_gdc() are all inside 001_initial_schema.sql. The pg_cron job is scheduled to run at 8AM UTC (2AM CT) automatically. Verify it worked the morning after running the schema.
Phase 03
Make.com Automation
Status: SPECIFIED — scenarios need to be built in Make.com · $9/mo
Four Make.com scenarios wire together your data sources and downstream systems. Build them after Phase 4 (Vercel) is live so the API routes exist as webhook targets.
Phase 04
Vercel Deployment
Status: PARTIAL — app live at fsos-seven.vercel.app · env vars not set
The app is deployed and serving at fsos-seven.vercel.app. The build is clean (type-check, lint, production build all pass). The only remaining task is setting the 10 environment variables in Vercel and redeploying. That one action unblocks Phases 5 and 6 simultaneously.
This is the current blocker. The command center loads but shows "Loading…" indefinitely because NEXT_PUBLIC_SUPABASE_URL is not set. Set env vars → redeploy → the full UI renders.
Phase 05
Forms + FNA System
Status: BUILT — all 3 API routes deployed · untestable until Phase 4 env vars set
All three API routes are deployed and correct. /api/forms/submit receives client responses. /api/forms/send delivers form links via Resend email and GHL SMS. /api/forms/fna calls Claude Sonnet to generate professional FNA reports. All will work the moment Vercel env vars are set and Supabase schema is running.
Phase 06
Agency Owner System
Status: BUILT — referral + upload pages deployed · untestable until Phase 4
The agency referral page (/[slug]), upload page (/upload/[slug]), and both API routes are deployed. Each agency owner gets two links — that is the complete interface they need. No login, no training, no portal. Everything else happens inside your system.
Phase 07
GHL Pipelines + AI Agents
Status: SPECIFIED — webhook handler deployed · GHL not yet configured
The GHL webhook handler is deployed at /api/webhooks/ghl. It handles AppointmentBooked (auto-sends forms), OpportunityStageChanged (auto-creates commission cases), ContactDNDUpdated (logs opt-outs), and ContactCreated (upserts to Supabase). Configure GHL to point at your Vercel URL and the pipeline activates.
AI Employee requires DM approval. Confirm your DM will enable GHL AI Employee on your sub-account before starting the AI agent configuration steps.
Phase 08
Claude Chrome Automation
Status: NOT STARTED — requires all prior phases complete
Install Claude in Chrome, record the APEX BOB export workflow, and schedule it for Monday 7AM. This eliminates your only remaining manual weekly task — the APEX export. When complete, the system runs entirely on its own.
